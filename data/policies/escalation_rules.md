# Escalation Rules

## Overview
This document outlines when and how support tickets should be escalated.

## Escalation Criteria
- P0 issues: Immediate escalation to on-call engineer
- P1 issues: Escalate within 1 hour if not resolved
- Security incidents: Immediate escalation to security team
- Billing disputes over $1000: Escalate to billing manager
- Account lockouts affecting multiple users: Escalate to identity team lead

## Escalation Paths
1. **Technical Issues**: Support → Engineering → Engineering Lead
2. **Billing Issues**: Support → Billing → Billing Manager
3. **Security Issues**: Any → Security Team → CISO
4. **Account Issues**: Support → Identity Team → Identity Lead

## Documentation
All escalations must include:
- Original ticket reference
- Reason for escalation
- Current status and attempted solutions
- Impact assessment

## Response Times
- P0: Immediate
- P1: Within 1 hour
- P2: Within 4 hours
- P3: Within 24 hours
